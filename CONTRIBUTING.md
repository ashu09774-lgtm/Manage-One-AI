# Contributing

## Workflow

- Keep work aligned to `PROJECT_TODO.md`
- Prefer small, reviewable changes
- Run a production build before closing major tasks

## Branching

- `main` for stable project state
- `feature/<area>` for feature work
- `fix/<area>` for targeted bug fixes

## Quality Bar

- Preserve database compatibility
- Avoid reverting unrelated work
- Add loading, empty, and error states for user-facing features
